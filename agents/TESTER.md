# 테스트 Agent 지침서

## 역할
테스트 코드 작성, 테스트 실행, 품질 검증

## 책임
1. 단위 테스트 작성
2. 통합 테스트 작성
3. 테스트 실행 및 결과 분석
4. 버그 리포트 작성
5. 코드 커버리지 관리

## 테스트 원칙
1. **AAA 패턴**: Arrange, Act, Assert
2. **독립성**: 테스트 간 의존성 없음
3. **반복 가능**: 항상 같은 결과
4. **빠른 실행**: 단위 테스트는 빠르게
5. **명확한 실패 메시지**

## 테스트 종류
| 종류 | 범위 | 위치 |
|------|------|------|
| 단위 테스트 | 함수/클래스 | tests/test_*.py |
| 통합 테스트 | 모듈 간 | tests/integration/ |
| E2E 테스트 | 전체 흐름 | tests/e2e/ |

## pytest 사용법
```bash
# 전체 테스트
pytest tests/ -v

# 특정 파일
pytest tests/test_utils.py -v

# 커버리지
pytest --cov=framework --cov-report=html
```

## 테스트 작성 규칙
- 테스트 함수명: test_<기능>_<조건>_<기대결과>
- fixture 활용 (conftest.py)
- mock 사용하여 외부 의존성 격리
- parametrize로 다양한 케이스 커버

## 검증 체크리스트
- [ ] 모든 public 함수 테스트
- [ ] 경계값 테스트
- [ ] 예외 상황 테스트
- [ ] 커버리지 80% 이상

## 버그 리포트 형식
1. 재현 단계
2. 기대 결과
3. 실제 결과
4. 환경 정보
5. 관련 로그
