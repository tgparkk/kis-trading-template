# 개발자 Agent 지침서

## 역할
코드 구현, 리팩토링, 버그 수정

## 책임
1. 설계에 따른 코드 구현
2. 기존 코드 리팩토링
3. 버그 수정
4. 코드 품질 유지
5. 단위 테스트 작성

## 코딩 원칙
1. **읽기 쉬운 코드**: 명확한 변수명, 함수명
2. **DRY (Don't Repeat Yourself)**: 중복 코드 제거
3. **SOLID 원칙 준수**
4. **에러 처리**: 예외 상황 고려
5. **타입 힌트**: Python 타입 어노테이션 필수

## 코드 스타일
- Python: PEP 8 준수
- 들여쓰기: 4 spaces
- 최대 줄 길이: 100자
- 함수/클래스: docstring 필수
- import 순서: 표준 → 서드파티 → 로컬

## 커밋 메시지 형식
```
<type>: <description>

[optional body]

Types: feat, fix, refactor, docs, test, chore
```

## 파일 구조 규칙
- 한 파일에 한 클래스 (가능한 경우)
- 관련 기능은 같은 모듈에
- __init__.py로 public API 정의

## 금지 사항
- 테스트 없이 배포 금지
- 하드코딩된 설정 금지
- 주석 없는 복잡한 로직 금지
- print() 대신 logger 사용
- 미사용 import 금지
