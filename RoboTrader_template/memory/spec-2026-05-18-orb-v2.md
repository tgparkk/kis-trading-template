### 외부 데이터 의존성
1. **전일 일봉 거래량** — 시나리오 시작 시 종목 리스트 × 168일 일괄 조회 → dict 캐시
   - **소스 우선순위**: `daily_candles` 테이블 (executor가 존재 확인). 미존재/결손 종목은 `minute_candles`에서 일별 sum(volume) fallback
2. **KOSPI 일봉 (시장환경 프록시)** — `daily_candles`에서 시나리오 전 구간 1회 일괄 조회
   - **최종 결정 (2026-05-19 점검)**: minute_candles에 KOSPI/ETF 분봉 모두 0건. 대안으로 `daily_candles`의 `KS11` (1,272행, 2021-03-04~2026-05-18) 사용.
   - **소스**: `daily_candles WHERE stock_code='KS11'`. 컬럼: `stck_bsop_date`(YYYYMMDD 문자열), `stck_clpr`(종가 문자열). 종가는 float 변환 필요.
   - **시장환경 정의 (단순화)**: `kospi_market_up = (직전 거래일 KS11 종가 > 그 전 거래일 KS11 종가)`. True이면 진입 허용, False이면 차단. 분봉 forward-fill / lookup 필요 없음.
   - dict: `{date_str: {"kospi_market_up": bool}}` — 일자별 단일 bool 플래그
   - look-ahead 방지: 당일 데이터(`stck_bsop_date == trade_date`)는 사용하지 않고 **직전/전전 거래일만 사용**

### 전략 본체 로직 (의사코드)
```python
def generate_signal(self, stock_code, data, timeframe):
    # 기존 ORB v1 게이트 (시간 + 박스 완성 + 돌파)
    if not breakout_detected(): return None

    # 필터 1: 거래량 (당일 누적 / 전일 일봉 비율)
    cum_vol_today = data["volume"][:cur_idx+1].sum()
    prev_day_vol = self._daily_ctx["prev_day_volume"].get(stock_code, 0)
    if prev_day_vol > 0:
        ratio = cum_vol_today / prev_day_vol
        if ratio < self._volume_ratio_threshold: return None
    # prev_day_vol==0 → fallback: 필터 미적용 (WARNING 로그)

    # 필터 2: 시장환경 (전일 KOSPI 일봉이 상승했을 때만 진입)
    if self._use_market_filter:
        flag = self._daily_ctx.get("kospi_market_up")
        if flag is None:  # 데이터 결손 fallback (전전일 KS11 결손 등)
            pass  # 통과
        elif flag is False:
            return None  # 전일 KOSPI 하락 → 진입 차단

    return self._make_buy_signal(...)
```

## §2. 그리드 & 평가 설계

### 기간 분할
- 전체: 2025-09-01 ~ 2026-05-15 (skip 202603)
- IS = 앞 70% ≈ 118일 (2025-09-01 ~ 2026-02-28)
- OOS = 뒤 30% ≈ 50일 (2026-04-01 ~ 2026-05-15)
- 토너먼트 결과 parquet에서 일자별 PnL 추출 → 사후 분할 (재실행 불필요)

### 파라미터 그리드 (96 시나리오)
| 파라미터 | 값 |
|---|---|
| `volume_ratio_threshold` | 0.5, 1.0, 1.5, 2.0 |
| `use_market_filter` | False, True |
| `max_positions` | 3, 4, 5 |
| SL / TP | (0.02, 0.04), (0.02, 0.06), (0.03, 0.04), (0.03, 0.06) |
| `box_minutes` | 30 (고정) |
| universe | dynamic (고정, top_n=50, volatility_pct) |

총 = 4 × 2 × 3 × 4 = **96 시나리오**, 예상 시간 ~2~3h (workers=8)

### 합격 조건 (3중 게이트)
1. **IS 합격선**: 일수익률 ≥ 0.3% AND 일승률 ≥ 50% AND MDD ≥ -15%
2. **OOS 합격선**: 같음
3. **일관성 게이트**: `|IS_daily - OOS_daily| / |IS_daily| < 0.30`

### 우승자 선정
- 3중 게이트 모두 통과한 시나리오 → OOS 종합점수 1위
  - `composite = 0.4×z(OOS_daily) + 0.3×z(OOS_win) + 0.3×z(OOS_calmar)`
- 0개 통과 → ORB 폐기 + 분봉 보류 결재 / 1+ 통과 → Phase 4 페이퍼 결재

## §3. 에러 처리 & 테스팅

### Fallback
| 결손 | 처리 |
|---|---|
| 전일 일봉 거래량 = 0 또는 미존재 | 해당 필터 미적용, 통과시킴, WARNING 로그 (종목 단위 1회만, 같은 종목 반복 로그 금지) |
| 직전/전전 거래일 KS11 일봉 누락 (kospi_market_up 미산출) | ctx에서 키 누락 → use_market_filter=True여도 통과, WARNING 로그 (일자 단위 1회만) |
