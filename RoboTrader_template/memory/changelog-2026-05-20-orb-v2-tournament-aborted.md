>
> 이 문서의 ORB v2 32 시나리오 결과(-48~-91%)는 **분봉 백테스트 엔진 버그로 오염**되었습니다.
>
> **버그**: `BacktestEngine.run_minute()`의 `trail_pct` 기본값이 `0.005`(0.5% 트레일링 스톱)이고, `scripts/run_intraday_tournament.py`가 이 값을 명시적으로 넘기지 않아 — 이 ORB v2 토너먼트 전체가 0.5% 트레일링 스톱을 강제로 뒤집어쓴 채 실행됨.
>
> **영향**: 분봉당 변동폭 ~0.2% 종목에서 0.5% 트레일은 +4~6% TP를 도달 불가능하게 만들어 R:R을 파괴. A/B 진단(2026-05-20): trail OFF 시 분봉 전략들이 -50~-81%에서 일부 breakeven 근처까지 회복(vwap_trade 8거래일 -21%→-1%). 본문 표의 PnL/win/MDD는 trail 오염값.
>
> **결론**: 본문 §90-93의 **"분봉 ORB 자체 폐기 확정"**, **"거래량+시장환경 필터 가설 기각"**, **"분봉 전략 전체 보류 권고"**는 모두 **무효/보류**. 4회 토너먼트(5/16·5/17·5/18·5/20)가 합격 0건이었던 것은 공통적으로 trail 버그를 공유했기 때문 — 전략 실패의 증거가 아님.
>
> **여전히 유효**: 과매수(고회전) 문제 — trail-OFF에서도 ma_trend 44거래/일 등 거래 회전 과다는 별개 이슈로 남음.
>
> trail 버그 수정 완료(2026-05-20: `run_minute` 기본값 0.005→None, `--trail` 옵션 추가). ORB 포함 trail-OFF 재토너먼트 진행 중. 상세: [research-2026-05-20-daytrading-deep-dive.md](research-2026-05-20-daytrading-deep-dive.md).

- 완료 32개 모두 PnL 음수, win rate < 33% (BEP 미달), MDD < -15%

### 2. 거래량 필터(vr05 → vr10) 효과 미미
- vr05_nomkt trades ~4,640 → vr10_nomkt trades ~3,608 (22% 감소)

### 3. 시장환경 필터(mkt OFF → ON) 효과 약간 (그러나 합격선 못 넘김)
- vr05_nomkt → vr05_mkt: trades 31% 감소, PnL 평균 6~10%p 개선
