### A.2 핵심 방법론 — 평균모멘텀스코어 (정량 룰로 환원)
**자산 성격**: **개별주 선별이 아니라, 위험자산(주식/주식ETF/지수)과 안전자산(현금/채권) 간 비중을 동적으로 조절**하는 마켓타이밍·자산배분.

1. **평균모멘텀스코어 계산**(자산 1개에 대해, 월말 기준):
   - 1·2·3···12개월 전 종가 대비 **현재가 ≥ 과거가**이면 각 1점, 아니면 0점 → 12개 신호.
   - `momentum_score = (1점 신호 개수) / 12` → **0 ~ 1 연속값**.
   - 예: 1·2·3개월만 매수신호 → 3/12 = 0.25.
2. **비중 결정(동적 자산배분)**:
   - 위험자산 비중 = `momentum_score` (예 0.25 → 주식 25%).
   - 안전자산 비중 = `1 - momentum_score` (예 75% 현금/채권).
3. **리밸런싱 주기**: **월간(매월말)**. 매월 스코어 재계산 → 비중 재조정.
4. **12개월을 쓰는 이유(저자 논지)**: 어느 룩백이 미래에 먹힐지 모르므로 1~12 전부 평균 → 단일 룩백 과최적화 회피·완만한 비중 변화(거래 빈도·whipsaw 완화).
5. **확장형(자료실/매뉴얼)**:
   - **다자산 평균모멘텀스코어**: 여러 위험자산(국내주식/해외주식/채권/금/리츠 ETF) 각각 스코어 산출 → 스코어 가중 + 정규화로 자산군 배분.
   - **변동성 조절(risk parity / target-vol)**: 각 자산 비중을 변동성 역가중(저변동 자산 ↑) — "주식채권혼합 + 변동성 risk parity + 모멘텀비중"(intelliquant 사례).
   - **듀얼모멘텀(Antonacci 변형)**: 상대모멘텀(자산 간 순위) + 절대모멘텀(현금 대비) — systrader79는 Antonacci 한국판 부록에 "한국형 듀얼모멘텀" 기고.

### A.3 정량화 가능 / 불가 부분
- **정량화 가능(전부)**: 평균모멘텀스코어, 비중 산출, 월간 리밸런싱, 변동성 역가중, 듀얼모멘텀 순위 — 100% 룰 기반. 정성 판단 없음(시스템 트레이딩 철학).

## 0. 설계 원칙

1. **systrader79는 자산배분/모멘텀 마켓타이밍** → 기존 개별주 횡단면 인프라(`_base_book_strategy.py`/`book_backtester.py`)는 **재사용 불가**(§2 상세). **신규 시계열 모멘텀 포트폴리오 백테스터**가 필요.
2. **핵심 명제 검증**: "평균모멘텀스코어 동적 자산배분이 KOSPI 단순보유 대비 MDD를 크게 낮추고 위험조정수익(Sharpe/Calmar)을 높이는가" — 저자의 "낮은 MDD 우상향" 테제를 한국 데이터로 검증/반박.
3. **데이터 현실 우선**: DB에 자산군 ETF 시계열이 **없다**(§7). 최소 실행안(KOSPI 단일 위험자산 + 현금 안전자산)부터 시작, 다자산은 ETF 백필 결정에 종속.
4. **월간 리밸런싱·연속 비중**: 종목 이진선택이 아니라 포트폴리오 비중 시계열 → equity curve를 직접 합성(개별 계좌 합산 ✗).
5. **거래비용**: 리밸런싱 회전율 기반(월 1회, 비중 변화분 × 왕복비용). 개별주 세금(0.18%)은 ETF면 비적용(ETF 매도세 면제) — 자산군에 맞춰 조정.
6. **no-lookahead**: 월말 t의 비중은 t 시점까지의 종가만으로 산출 → t+1월 보유. (펀더멘털 PIT lag 대신 **월말 신호→익월 보유** 시점규약.)

systrader79 코드화의 병목은 **방법론이 아니라 자산군 시계열 데이터**다. DB 실측(§7):

| 단계 | 자산 유니버스 | 백필 필요? | 비고 |
|------|--------------|-----------|------|
| **MVP(즉시)** | KOSPI(위험) + 현금(안전) | **불필요** | 평균모멘텀스코어 코어 검증. 1자산 동적 노출. |

## 1. 신호 (평균모멘텀스코어)

자산 A의 월말 종가 시계열 `P[m]`(월말 m)에 대해:
```
score_A[m] = (1/12) * Σ_{k=1..12} 1[ P[m] >= P[m-k] ]      # 0 ~ 1 연속
```
- 룩백 단위 = **월**(월말 종가). 일봉 DB → 월말 리샘플(`resample('M').last()` 또는 영업일 월말).
- 워밍업: 최소 12개월 히스토리 필요(2021-01 시작이면 2022-01부터 신호 산출 가능; KOSPI 65개월 → 약 53개월 백테스트 구간).
- 다자산: 각 자산 score 독립 산출.

## 2. 비중 산출 (포트폴리오 — 기존 인프라와 불일치 지점)

**MVP(1위험자산)**:
```
w_risk[m]  = score_KOSPI[m]
w_safe[m]  = 1 - w_risk[m]
```
**다자산(score 가중 정규화)**:
```
w_i[m] = score_i[m] / Σ_j score_j[m]   (Σscore>0; 전부 0이면 100% 안전자산)
```
**변동성 조절(옵션)**: `w_i ∝ score_i / vol_i` (vol_i = 최근 N개월 수익률 표준편차) 후 정규화.
**듀얼모멘텀(옵션)**: 절대모멘텀(score>임계, 예 0.5) 통과 자산만 후보 → 상대모멘텀 상위 K개 균등/score가중.

> **여기가 기존 `book_backtester`와 갈라지는 핵심**: 출력이 "종목 리스트"가 아니라 **자산별 연속 비중 벡터 w[m]**. `BookStrategy.generate_signal`은 종목당 BUY/HOLD 이진 Signal만 반환 → 비중 표현 불가. `BookBacktester.run_single`은 한 종목 단일 포지션 풀인/풀아웃만 → **부분 비중·리밸런싱 표현 불가**. 재사용 불가 확정.

## 3. 신규 백테스터 명세 (필수 신규 인프라)

```
backtest/allocation_backtester.py   # 신규
  - 입력: assets = {asset_code: monthly_close_series}, safe_asset (code or cash_rate)
  - 월말 루프: score → weight vector w[m] → 익월 보유
  - equity[m+1] = equity[m] * (1 + Σ_i w_i[m] * ret_i[m→m+1] + w_safe[m]*safe_ret)
  - 리밸런싱 비용: turnover[m] = Σ_i |w_i[m] - w_i_drift[m]| ; cost = turnover * round_trip_bps
  - 산출: equity_curve(월간), CAGR, Sharpe(월→연 환산 √12), Sortino, MaxDD, Calmar, 위험자산 평균노출
  - 벤치마크: KOSPI b&h equity 동시 산출 → 알파/베타/상관/하락장 캡처비율
```
- **월간 빈도** 메트릭(기존 `_compute_metrics`의 분봉 `√(252*390)` 부적합 → `√12`). 기존 `book_backtester._compute_metrics` 재사용 ✗(연율화 상수 다름).
- no-lookahead: w[m]은 P[..m]만 사용, 수익은 m→m+1.

## 4. 청산/리밸런싱 Variant
자산배분은 "청산(sl/tp/trail)" 개념이 약함(상시 비중 보유). 대신 **리밸런싱·비중 변형**:
| Variant | 설명 |
|---------|------|
| **M (코어)** | 평균모멘텀스코어 = 비중, 월간 리밸런싱, 안전자산=현금0% |
| **MB** | 안전자산=채권ETF(백필 시), 월간 |
| **MV** | 변동성 역가중 추가(target-vol) |
| **MD** | 듀얼모멘텀(절대>0.5 게이트 + 상대순위 top-K) |
| **BH** | KOSPI 100% 단순보유(벤치마크) |

> `_base_book_strategy.BookStrategy`를 **상속하지 않음**(인터페이스 불일치). 별도 `Allocator` 추상으로.

**옵션 B(비권장 — 억지 끼워맞춤)**: KOSPI 단일자산을 "1종목"으로 보고 `book_backtester`에 score>임계 시 풀인/풀아웃(이진). → 연속 비중 손실, 저자 방법론의 핵심(부분 노출·완만한 비중)이 사라짐. **방법론 왜곡이라 권장 안 함.**

## 8. 검증
- **벤치마크 대비**: KOSPI b&h equity와 동시 산출 → CAGR/Sharpe/MaxDD/Calmar/베타/하락장 캡처비율. 저자 테제("낮은 MDD")를 MaxDD 절대비교로 검증.
- **국면 분할**: 2021~2026 BULL/BEAR/SIDEWAYS(기존 `regime_split_*` 라벨 재사용)에서 위험자산 평균노출·하락장 방어 측정 — Elder 포트폴리오 결론("시장중립 방어형")과 대조.
- **no-lookahead**: w[m]이 P[..m]만 참조(월→익월 시점규약) 단위테스트.
- **회전율/비용 민감도**: round_trip_bps 0/15/30bp 스윕.
- **워밍업 손실**: 12개월 워밍업으로 백테스트 구간 ≈53개월(2022-01~2026-05) — 짧음. 한계 명시.

## 9. 사장님 결정 필요 (코드화 전 결재)

1. **자산 유니버스 범위** — (a) **MVP: KOSPI(위험)+현금(안전) 단일자산** 즉시 코드화(백필 0, 저자 코어 검증) / (b) **2자산: KOSPI+채권ETF**(ETF 1종 백필) / (c) **다자산: 국내·해외주식·채권·금·리츠 ETF 5~7종**(백필 다수, 듀얼모멘텀·risk parity 완전체). **권장 = (a)로 코어 즉시 검증 → 효과 확인 후 (b)/(c) 단계 확장.**
2. **ETF/자산군 데이터 백필 여부** — 다자산을 하려면 ETF 일봉 백필 선행(FDR `DataReader('069500'...)` `backfill_kospi_index.py` 패턴 재사용). 상장일·거래량·괴리 검증 필요. **결정: 백필할 ETF 목록·시작연도(2021 vs ETF 상장일)·예산.** (KOSDAQ 지수도 동일.)
3. **신규 백테스터 채택** — 자산배분은 기존 `book_backtester`(종목별 독립계좌·단일포지션)와 **구조 불일치 → `backtest/allocation_backtester.py` 신규 필요**. **결정: (a) 신규 자산배분 백테스터 신설(권장) / (b) book_backtester에 이진 풀인/풀아웃으로 억지 끼워맞춤(방법론 왜곡, 비권장).**
4. **코드 배치 트랙** — systrader79는 `strategies/books/`가 아니라 **`strategies/allocation/` 신규 트리**가 개념상 맞음(개별주 책이 아님). **결정: 신규 allocation 트랙 / 기존 books 트랙 내 예외 배치.** (권장 = allocation 신규.)
5. **안전자산 정의** — 현금 0% vs 고정캐리(연 N%) vs 실제 채권ETF. MVP는 현금 0% 권장(보수적).
6. **벤치마크/책간 비교** — 개별주 책들과 **직접 A/B 불가**(대상·계좌모델 상이). 비교는 KOSPI b&h 및 Elder 포트폴리오 "방어형 분산자산" 결론과의 **정성적 대조**로 한정. 리포트에 명시.

## 10. 한계 (전면)
- **책 실물 미보유** — 블로그/wikidocs/인터뷰 재구성. 평균모멘텀스코어 공식(12룩백·월간·score=비중)은 다출처 일치하나, 저자의 정확한 안전자산·다자산 가중·변동성조절 세부 파라미터는 실물/자료실 확인 시 미세조정 필요.
- **데이터가 최대 병목** — DB에 자산군 ETF/채권/금/달러/KOSDAQ **전무**. MVP(KOSPI+현금) 외 모든 변형은 백필 선행. 저자 방법론의 진가(다자산 분산·하락장 채권 헤지)는 백필 없이는 부분 재현 불가.
- **기존 인프라 비호환** — `_base_book_strategy`/`book_backtester`(종목별 독립계좌·횡단면 순위·이진 신호)는 연속 비중 자산배분을 표현 못 함. 신규 `allocation_backtester` 필수(약 200~300줄 신규 코드 추정). **재사용률 낮음(인프라 측면 ~10%, FDR 백필 패턴·regime 라벨·리포트 유틸 정도만 재활용).**
- **짧은 히스토리** — KOSPI 65개월 − 12개월 워밍업 = ≈53개월 백테스트. 단일 국면(2021~2026, 특히 2025~26 대폭등 편향) 위험. 다년·다국면 강건성은 표본 한계로 약함.
- **개별주 알파와 분리** — 본 트랙은 마켓타이밍/자산배분이라, 그간 축적한 개별주 펀더멘털·기술적 책들과 포트폴리오 통합 시 별도 메타(주식 슬리브에 systrader79 비중 게이트 적용 등) 설계 필요 — 본 설계 범위 외.
