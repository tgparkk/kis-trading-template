# Phase 0 — 피처 엣지 랩 (Feature Edge Lab) 설계

> 작성: 2026-06-13 · 상태: 설계 확정(사장님 승인) → 구현 계획 대기
> 목적: "우리만의 파생 데이터(피처)"가 실제 예측 엣지를 갖는지 **측정만** 하고, 그 증거로
>       향후 방향(A 전략별 필터 / B 횡단면 스코어 / C 시장 게이트)을 결정한다.

## 1. 배경 / 동기

현 8개 페이퍼 전략은 **가격·거래량 단일 축**(일봉 OHLCV + 시총 + 실시간 지수 급락가드)만 쓴다.
프로젝트 DB에는 안 쓰고 있는 데이터가 있으며, 이번에 일부를 백필했다. 사장님 지시는
"우리만의 데이터를 만들어 예측에 쓰고 싶다 + 우선 탐색·측정만"이다.

과적합을 피하기 위해 **모델 학습·라이브 배선·신규 전략 생성 이전에**, 후보 피처들이
선행수익률/전략신호 결과를 실제로 가르는지부터 증거기반으로 측정하는 단계를 둔다.

## 2. 범위 (YAGNI)

**포함**
- PIT 피처 패널 빌드 → parquet 적재(사장님 확정: parquet).
- 피처 엣지 측정 = (a) 전 패널 대상 선행수익률 예측력(IC·십분위 스프레드) + (b) 8전략 과거
  진입신호의 결과 조건부 기대값.
- 부트스트랩·OOS·커버리지를 갖춘 랭킹 리포트.

**제외 (Phase 0 아님)**
- ML 모델 적합/학습. 라이브 배선. 신규 전략 등록. 기존 전략 파라미터 변경.
- VKOSPI(내재변동성): 무료 소스 부재로 백필 불가 → **지수 실현변동성으로 대체**, 소스 확보 시 후속.

## 4. 아키텍처 / 컴포넌트

신규 모듈 `scripts/feature_edge/` (각 1책임, 독립 테스트 가능):

1. **`feature_panel.py` — PIT 피처 패널 빌더**
   - 입력: 유동성 플로어(거래대금 ≥ 10억) 통과 (stock, date) 전체, 2021-2026.
   - 출력: `reports/discovery/feature_edge/feature_panel.parquet` (인덱스 = stock_code × date).
   - 모든 피처는 **의사결정 시점 가용 데이터만** 사용(§6 PIT 규약).
   - `QuantDailyReader`(클린 일봉) 재사용. adj_factor 곱셈 금지.

2. **`labelers.py` — 결과 라벨러**
   - (a) **선행수익률 라벨**: 각 (stock, date)의 T+1 시가 진입 가정 forward 5/10/20일 수익률.
   - (b) **트리플배리어 라벨**: H일 내 +X% 선도달(win) / −Y% 선도달(loss) / 시간초과. (X/Y/H는
     측정용 표준값 세트, 전략 청산과 별개).
   - (c) **전략신호 라벨**: 8전략 진입룰을 **백테스트 단일소스**(`strategies/books/**/rules*.py`,
     `scripts/rs_leader/rule.py`, `scripts/discovery/rules.py`)로 재생성 → 각 전략 **실제 청산
     어댑터**로 트레이드 결과(수익률·승패·MFE/MAE) 라벨. 라이브와 동일 룰 보장.

3. **`edge_metrics.py` — 엣지 측정 엔진**
   - 연속 피처 vs 선행수익률: 일별 **Spearman IC** → IC 평균·IC IR(평균/표준편차)·십분위 스프레드.
   - 피처 vs 전략신호 결과: 터사일/사분위 분할 승률·평균 트레이드수익·기대값, 피처 vs 트레이드 PnL IC.
   - **부트스트랩 CI**(블록 부트스트랩, p05) + **Train/OOS 홀드아웃**(train 2021~2024.6 /
     test 2024.7~2026.5) 부호 일치 검사 + **커버리지**(피처 non-null 비율) 병기.

4. **`run_edge_lab.py` — 오케스트레이터**
   - 패널 빌드 → 라벨 → 측정 → 리포트. `--stage {singles,interactions}` 2단계 분리.
   - 산출: `reports/discovery/feature_edge/edge_report.md` + 보조 parquet/tsv.

테스트: `tests/feature_edge/` (PIT 룩어헤드, 라벨 정합, 메트릭 수치, 부트스트랩 재현).

## 5. 피처셋 v1 (확정)

전종목·5년 = ✓ / 좁은 커버리지 = △ / 신규데이터 = ⚡

**단일 피처**
- **추세·모멘텀 ✓**: returns_5d, returns_20d, MA20 이격, MA60 이격, MA20−MA60 정렬, 모멘텀 가속(returns_5d − returns_20d/4), 신고가 근접도(close/롤링고점)
- **변동성 ✓**: volatility_20d, 변동성 시계열·횡단면 백분위, 변동성 추세(vol_5d/vol_20d)
- **유동성·거래량 ✓**: 거래대금 수준, 거래량 급증배수(당일/20일평균), 거래대금 추세(5d/20d), Amihud 비유동성(|return|/거래대금)
- **수급 △⚡**: 외국인 순매수량/20일평균거래량(T-1), 외국인 누적순매수 5/20일, 연속 순매수 streak
- **시장·국면 ✓**(지수 자체계산): KOSPI/KOSDAQ MA20·MA60 추세, 지수 실현변동성 백분위(VKOSPI 대체), 시장폭(MA20 상회 종목%), 횡단면 수익 분산, 상대강도(종목 returns_20d − 지수 returns_20d)
- **이벤트 △⚡**: 증자/분할/관리종목 N일내 플래그, 이벤트까지 일수
- **일중파생 ✓⚡**(minute_candles): 시가갭(open/전일종가−1), 오프닝 30분 레인지·방향, 종가 일중위치((close−low)/(high−low)), VWAP 대비 종가, 장초반 거래량 집중도
- **펀더멘털 △**(조건부): PBR·PER 횡단면 랭크, ROE, 배당수익률

**교차항(2단계에서 측정, 단일 통과분 한정)**
1. 외국인 누적순매수 × MA20 눌림 (스마트머니+눌림목)
2. 종목 모멘텀 × 시장 risk-on(시장폭) (조건부 모멘텀)
3. 거래량 급증 × 종가 강함 (진짜 돌파 vs 페이크)
4. 저변동 × 양의 모멘텀 × MA60 위 (퀄리티 추세)
5. 가격 약세 × 외국인 매수 (다이버전스)
6. 임의 신호 × 이벤트-클린 (배제필터 기대값)

## 6. PIT(룩어헤드 0) 규약

- 일봉: 미확정 당일봉 제외(확정봉만), 신호/피처는 T 종가까지만.
- 선행 라벨: **T+1 시가 진입** 이후만 사용(진입봉 정보 라벨 누출 금지).
- 수급 `foreign_flow`: T일 마감 후 발표 → 피처는 **shift(1)** 로 T-1 참조.
- 재무 `financial_statements`: `report_date`(공시일) 이후에만 가용으로 취급(기간말 아님).
- 시장피처: 지수도 미확정 당일 제외, T 종가까지.
- 분봉 일중파생: 진입 의사결정이 익일(T+1)이면 T일 종료된 분봉만 사용.

## 7. 반-과적합 / 다중검정 프로토콜

- **2단계 측정**: 단일 피처 전수 측정 → 통과분만 교차항 측정(조합 폭발 금지).
- **판정 게이트**: 피처는 (부트스트랩 IC/기대값 **p05 > 0**) ∧ (**OOS 부호 유지**) ∧
  (**커버리지 ≥ 임계**, 기본 60%) 를 모두 만족해야 "엣지 후보".
- **다중검정 주의**: 측정 피처 수와, 우연 통과 기대 수(예: BH-FDR 또는 보수적 p05 강화)를 리포트에 명시.
- 측정 전용 — test 셋에 피처선택·튜닝 금지.

## 8. 산출물 / 성공 기준

- `reports/discovery/feature_edge/feature_panel.parquet` — 재사용 피처 패널.
- `reports/discovery/feature_edge/edge_report.md` (+ 보조 parquet/tsv) — 피처별 IC·IC IR·
  십분위/터사일 스프레드·부트스트랩 p05·OOS 부호·커버리지를, **전 패널 + 전략별 + 풀드**로 랭킹.
- **결정 섹션**: 측정 결과 → 다음 방향 권고.
  - 전략신호 조건부 엣지가 강함 → **A(전략별 메타필터)**.
  - 횡단면 선행수익률 IC가 강함 → **B(컴포짓 스코어 신규전략)**.
  - 시장피처가 포트폴리오 worst-regime을 가름 → **C(예측형 시장 게이트)**.
- 성공 = "측정이 끝나 A/B/C를 **가정이 아닌 증거로** 고를 수 있는 상태".
