# 전략 발굴 파이프라인 (Strategy Discovery Pipeline) — 설계

날짜: 2026-06-11 · 상태: DRAFT (사장님 승인 대기) · 구현 난이도: 중 (기존 MV4 부품 ~90% 재사용)

## 0. 목적과 비목적

**목적**: 신규 전략 후보를 **생존 필터**로 거르는 표준화·반복 가능한 발굴 공정.
목표는 "월 X% 전략 찾기"가 아니라 **"기존 포트폴리오의 합성 Sharpe를 올리는 저상관 엣지의 누적"**이다.
(근거: MULTIVERSE4 — 현 7전략 상관 0.83=단일베타, 합성 Sharpe 0.5 < KOSPI 0.99. 합성 Sharpe를 올리는 유일한 정석 경로는 저상관 전략 추가.)

**비목적 (금지 사항)** — 이 레포에서 실증된 실패 경로의 재진입 차단:
- ❌ **목표수익률 역산 튜닝**: 임계값을 수익률 목표에 맞춰 조정 (1·2차 멀티버스에서 2회 과적합 판정)
- ❌ **in-sample best-pick**: 그리드 스윕의 최고 조합 채택 (그리드는 강건성 *확인*용)
- ❌ **분봉 일중 단타 재탐색**: 19권 연구에서 백테스트 가능한 전 영역 전멸 (-22%~-99%), 체결 우위 부재
- ❌ **층 분리 측정**: 다층 시스템(후보선별→실행)은 반드시 체이닝 측정 (Book19 교훈)

## 1. 채택 게이트 G0~G6 (순차, 하나라도 FAIL이면 기각·사유 기록)

### G0 — 후보 소싱 (저상관 우선 쿼터)
아이디어 인테이크 시 카테고리를 명시하고, **추세추종 계열은 배치당 1개 이하**로 제한:

| 카테고리 | 저상관 근거 | 데이터 가용성 |
|---|---|---|
| 단기 과락 평균회귀 (연속음봉/과락 후 반등) | 추세와 반대 방향 진입 | quant 일봉 ✓ |
| 오버나이트 갭 (종가 매수→익일 시가/장중 매도) | 보유 시간대 비중첩 | quant 일봉 ✓ |
| KOSDAQ↔KOSPI 상대 스위칭 / 섹터 로테이션 | 횡단면 중립 | quant 일봉+지수 ✓ |
| 인버스/현금 헤지 오버레이 | 상관 음수 (전략이 아닌 레이어) | KOSPI 일봉 ✓ |
| 캘린더 효과 (월말월초·요일) | 가격 패턴과 무관한 원천 | quant 일봉 ✓ |
| 수급 기반 (외인/기관 순매수) | 가격 외 정보 원천 | **✗ 데이터 부재 확인**(quant DB 테이블 없음, 2026-06-11) — 수집 선행 없이는 후보 불가 |

기존 미검증 자산 재활용 우선: `strategies/mean_reversion/`, `strategies/bb_reversion/` 템플릿 샘플은 정식 게이트를 통과한 적 없음 → 첫 배치 후보로 무비용 재활용 가능.

### G1 — 룰 코드화 (verbatim + TDD)
- `Rule` 클래스 1개 (기존 books 패턴), 파라미터는 출처 사양 그대로 (임의 튜닝 금지)
- no-lookahead 테스트 필수 (미래봉 절단 불변성), NaN/워밍업 가드 테스트
- 청산 정의 필수: 진입만 있고 청산이 재량이면 G1에서 기각 (18·19권 교훈 — 측정 불가)

### G2 — 1차 백테스트 (MV4 파이프라인 등록)
`scripts/multiverse4_returns_export.py` 의 `SPECS` 에 후보 등록 → 풀기간(2021~) 연속 백테스트 → daily_returns CSV.
- 사이징: **라이브 동일** (현행 종목당 100만 고정; 사이징 일원화 결정 시 스위치 1곳)
- 청산 충실도: 라이브에 올릴 청산과 동일 어댑터 (sl/tp/mh 또는 MA trail — 기존 어댑터 재사용)
- **컷**: 풀기간 PnL > 0 · Sharpe > 0.4 · 청산거래 ≥ 100건 (표본 미달이면 "미결" 보류, 기각 아님)

### G3 — ★상관 게이트 (이번 설계의 핵심 신규)
기존 활성 전략 합성(live_sum)·각 단독과 비교:
- **합성과 피어슨 상관 < 0.5** AND **꼬리 동시손실 lift < 2.5** (lift: 하위 10% 손실일 동행배수)
- 둘 중 하나라도 초과 → **Sharpe가 아무리 좋아도 기각** ("N번째 변주" 차단)
- 도구: `multiverse4_portfolio_analysis.corr_matrix` / `tail_coloss_lift` 그대로 재사용
- 부가 측정: 후보 추가 시 합성 Sharpe 증분 (ΔSharpe ≤ 0 이면 기각)

### G4 — 강건성 3축 (MV4 분석 재사용)
1. **워크포워드**: 반기 양수 ≥ 7/11 AND 최악 반기 > -15%
2. **부트스트랩**: 블록(21일)×1000회 Sharpe **p05 > 0**
3. **비용**: 편도 슬리피지 30bp 스윕에서 PnL > 0 (`--slippage 0.003` 재실행)

### G5 — 과적합 방어 2축
1. **파라미터 섭동**: 핵심 파라미터 ±20% 5점에서 PnL 부호 유지 (best-pick이 아니라 평탄성 확인)
2. **OOS 홀드아웃**: train(2021~2024.6) / test(2024.7~) **둘 다 Sharpe > 0** (가치3책 false-positive를 잡은 검증)

### G6 — 페이퍼 졸업 (G0~G5 전체 PASS 시에만)
- `strategies/<name>/` (strategy+screener+config) 작성, 격리 1천만, paper_trading=true
- **슬리피지 실측 모니터링 동시 가동**: 페이퍼 체결가 vs 신호시점가 괴리 누적 기록 (rs_leader 교훈 — 편도 10bp 초과 시 G4-③ 재평가)
- 8주 후 전달력 평가: 페이퍼 실측 vs 백테스트 동기간 (괴리 원인 = 체결/운영버그/국면)
- **leaderboard 등재는 페이퍼 실적 누적 후** (기존 원칙 유지)

## 2. 신규 구현 범위 (작음 — 기존 부품 조립)

| 신규 | 내용 | 규모 |
|---|---|---|
| `scripts/strategy_gate.py` | 후보명 받아 G2~G5 일괄 실행 → PASS/FAIL 게이트 리포트 출력 (mv4 export/analysis 함수 호출 + 섭동·OOS 루프) | ~300줄 |
| `reports/discovery/CANDIDATES.md` | 후보 대장: idea → coded → G2..G6 상태 + 기각 사유 (재탐색 방지 기록) | 문서 |
| `tests/test_strategy_gate.py` | 게이트 판정 로직 단위 테스트 (토이 returns 로 각 게이트 PASS/FAIL 경계) | ~150줄 |

재사용 (수정 없음): `multiverse4_returns_export.py`(SPECS 등록만), `multiverse4_portfolio_analysis.py`(corr/tail/bootstrap/walkforward 함수), `book_param_multiverse` 로더, `exit_multiverse` 어댑터·`run_portfolio`.

## 3. 운영 사이클

- **배치 단위**: 후보 3~5개 / 배치. G1(코드화)이 병목(후보당 반나절), G2~G5는 자동(후보당 ~10분)
- **기대치 관리**: 배치당 채택 0~1개가 정상. 기각이 기본값이고 기각 사유가 자산 (CANDIDATES.md 누적)
- **첫 배치 제안**: ①단기 과락 평균회귀(신규 룰) ②오버나이트 갭(신규 룰) ③`bb_reversion`(기존 템플릿 재활용) ④`mean_reversion`(기존 템플릿 재활용) — 전부 비추세 계열로 G3 통과 가능성 우선
- 모든 수치 보고서는 verifier 대조 (MV4 관례)

## 4. 선결/연계 과제 (파이프라인 밖, 별도 결정)

1. **사이징 일원화** (종목당 100만 고정 → 자본/K 균등): 결정 시 G2를 동일 스위치로 재측정. 파이프라인 자체는 현행 사이징으로 즉시 가동 가능
2. **국면 연동 배분 레이어**: 발굴과 직교하는 수익 레버 (강세장 베타 활용). 별도 멀티버스로 검증 — 본 설계 범위 밖
3. **ma20·ma5 페이퍼 정리**: 슬롯·관찰 노력 회수 (3축 전미달 확정)

## 부록 A — 배치 3 기획: 희소 과락 반등의 선별강도×엣지 곡선 (2026-06-11, 사장님 지시)

**가설(사전 선언)**: 과락의 희소성(연속하락 일수 N, MA 이탈 깊이)이 거래당 그로스 엣지를
단조 증가시키며, 어느 a priori 지점에서 생존선을 넘는다.
근거(배치 1·2 실측): N축 단조 개선 N1 -90.8% → N5 -11.4% (h1, top50), MA 이탈 -8%
+21.8% → -12% +32.2%. N=5 h1의 거래당 그로스 ≈ +0.33% (생존선 0.41% 근접).

**제약 C1 (사장님 지시)**: 풀기간 월평균 거래 ≥ 10회 (총 ≥650 청산). G2에 빈도 게이트 추가.
배치2 실측 빈도: top50 N=5 h1=월 21회·h2=월 14회 → 더 깊은 N은 **유니버스 top300 확장**
(rs_leader 선례)으로 빈도 보전. top300 하위 유동성의 슬리피지 과소평가 위험은
일평균 거래대금 하한으로 완화(로더의 거래대금 정렬이 자연 필터).

**측정 매트릭스 (사전 고정 — 결과 본 후 추가 금지)**:
- A. deep_down_bounce: N ∈ {4,5,6,7} × 보유 {h1,h2} × top300 = 8변형
- B. deep_ma_deviation: 이탈 {-12,-15,-20%} × top300, 청산=MAReversionExitAdapter = 3변형
- C. confluence: N=4 연속하락 AND 거래량 ≥2×(20일평균) × h2 = 1변형 (독립 조건 중첩)
- 각 변형 = 기본비용 + 무비용 쌍둥이 런 → **거래당 그로스/넷 엣지 곡선** 산출 (신규 계측기)

**판정 룰 (사전 선언)**:
1. **현물 채택**: 거래당 그로스 ≥0.5% AND 월 ≥10회 AND N축 단조성 유지 AND OOS(train/test)
   패턴 유지 AND ΔSharpe>0 AND G4(부트스트랩·비용30bp) 통과
2. **ETF 조건부**: 그로스 0.2~0.5% 구간 → 현물 기각·"거래세 면제 상품 재검 후보" 분류
   (ETF 생존선 ≈ 0.15~0.2%)
3. 그 외 기각
- **G3 수정(단기보유 한정, a priori 명시)**: 1~2일 보유는 in-market일 베타로 corr 0.8+가
  구조적(배치2 확인) → corr/lift는 참고치로 강등, **구속 기준 = ΔSharpe>0**(합성 개선 여부가
  G3의 목적인 "중복 변주 차단"을 직접 측정).

**정직 고지**: N∈{4..7} 선언은 배치2 그래디언트를 *본 후*의 선택이라 순수 a priori 아님 —
단조성·OOS 요건이 보정 장치이며, 채택 시 페이퍼(G6)에서 재검증이 최종 방어선.

## 5. 성공 기준 (파이프라인 자체의 KPI)

- 6개월 내: G3 통과(상관<0.5) 신규 전략 ≥ 1개 페이퍼 가동
- 합성(live_sum) 부트스트랩 Sharpe p05 가 +0.01 → +0.3 이상으로 상승 (분산이 실제로 생겼다는 통계적 증거)
- 기각 후보의 사유가 100% 기록·재현 가능 (CANDIDATES.md)
