# 추세 기반 적응형 청산 시스템 가이드

## 🎯 개요

기존의 **고정 익절/손절률**(예: 무조건 20% 익절, 10% 손절) 대신, **추세와 모멘텀을 분석**하여 동적으로 청산 시점을 결정하는 시스템입니다.

### 핵심 아이디어

```
기존 방식:
  수익률 20% 도달 → 무조건 매도 ❌ (더 오를 수 있는데 기회 놓침)
  수익률 5% → 계속 보유 ❌ (하락 전환해도 들고 있음)

개선 방식:
  수익률 20% + 추세 강함 → 보유 지속 ✅ (추세 꺾일 때까지)
  수익률 5% + 추세 약화 → 조기 청산 ✅ (수익 보호)
```

---

## 📊 작동 원리

### 1. **모멘텀 종합 점수 계산** (0~100점)

5가지 기술적 지표를 종합하여 현재 추세의 강도를 점수화합니다:

| 지표 | 가중치 | 설명 |
|------|--------|------|
| **이동평균선 정배열** | 30% | 5일 > 20일 > 60일 순서 확인 |
| **RSI 위치** | 25% | 50~70: 건강한 상승 (만점) |
| **거래량 추세** | 20% | 최근 5봉 vs 이전 20봉 비교 |
| **가격 패턴** | 15% | 연속 양봉/음봉 체크 |
| **MACD 신호** | 10% | 골든크로스/데드크로스 |

**점수 해석:**
- **70점 이상**: 강한 상승 추세 → 익절률 초과해도 보유
- **30점 이하**: 약한 추세/하락 전환 → 조기 청산
- **30~70점**: 수익률과 모멘텀 복합 판단

---

### 2. **트레일링 스톱** (Trailing Stop)

최고 수익률을 추적하여, 고점 대비 일정 비율 하락 시 청산합니다.

```python
# 예시
매수가: 10,000원
현재가: 12,500원 → 수익률 25% (최고점)

# 이후 하락...
현재가: 12,200원 → 수익률 22%
→ 최고점 25% 대비 -3% 하락 → 청산! ✅

# 결과: 22% 수익 확보 (20% 고정 익절보다 +2%)
```

**설정:**
- **트레일링 비율**: 3% (최고점 대비)
- **최소 수익률**: 5% (이상일 때 트레일링 활성화)

---

### 3. **추세 기반 청산 판단 로직**

#### Case 1: 강한 추세 (모멘텀 70+)
```
수익률 20% 도달
+ 모멘텀 75점
→ 보유 지속 ✅

이유: 아직 상승 여력 충분
```

#### Case 2: 약한 추세 (모멘텀 30-)
```
수익률 5%
+ 모멘텀 25점
→ 조기 청산 ✅

이유: 추세 약화 감지, 수익 보호
```

#### Case 3: 중간 추세 (모멘텀 30~70)
```
수익률 16% (익절 목표 20%의 80%)
+ 모멘텀 45점 (중립)
→ 청산 ✅

이유: 익절 근처 + 모멘텀 회복 없음
```

#### Case 4: 트레일링 스톱
```
최고 수익률 25%
현재 수익률 22%
→ 청산 ✅

이유: 고점 대비 -3% 하락
```

---

### 4. **긴급 청산 (안전장치)**

극단적 상황에서 강제 청산:

1. **-15% 손실**: 무조건 청산
2. **-12% 손실 + 모멘텀 20 이하**: 추세 붕괴 청산

## ⚙️ 설정 커스터마이징

### `config/trend_exit_config.json` 수정

```json
{
  "trend_based_exit": {
    "enabled": true,  // false로 변경하면 기존 방식 사용

    "trailing_stop": {
      "trailing_stop_pct": 0.03,        // 3% → 5%로 변경 가능
      "min_profit_for_trailing": 0.05   // 5% → 3%로 변경 가능
    },

    "momentum_thresholds": {
      "strong_trend": 70,  // 강한 추세 기준 (조정 가능)
      "weak_trend": 30     // 약한 추세 기준 (조정 가능)
    }
  }
}
```

### `trading_decision_engine.py`에서 활성화/비활성화

```python
# 초기화 시
self.use_trend_based_exit = True   # False로 변경 시 기존 방식 사용
```

## ❓ FAQ

### Q1: 기존 익절/손절률은 어떻게 되나요?
A: 여전히 **안전장치**로 작동합니다. 추세 분석이 실패하거나, 극단적 상황에서는 기존 방식이 적용됩니다.

### Q2: 트레일링 스톱 비율을 조정하고 싶어요.
A: `config/trend_exit_config.json`에서 `trailing_stop_pct` 값을 수정하세요.

### Q3: 모멘텀 점수가 낮게 나와요.
A: 시장이 약세이거나 횡보장일 가능성이 높습니다. 이 경우 조기 청산으로 손실을 방어합니다.

### Q4: 더 공격적으로 운용하고 싶어요.
A: `strong_trend` 기준을 70 → 60으로 낮추면 됩니다 (더 많은 종목을 보유 지속).

### Q5: 더 보수적으로 운용하고 싶어요.
A: `weak_trend` 기준을 30 → 40으로 높이면 됩니다 (더 빨리 청산).

## 🎯 요약

### Before (기존 방식)
```
고정 익절 20% → 무조건 매도
고정 손절 10% → 무조건 매도
```

### After (추세 기반 방식)
```
수익률 + 추세 강도 → 동적 판단
트레일링 스톱 → 최대 수익 보호
조기 청산 → 추세 약화 시 수익 확보
```

### 기대 효과
1. **더 큰 수익**: 강한 추세에서 20% 이상 수익 가능
2. **손실 방어**: 추세 약화 조기 감지로 손실 최소화
3. **수익 보호**: 트레일링 스톱으로 최대 수익 근처에서 청산
4. **안전장치**: 기존 방식이 폴백으로 작동
