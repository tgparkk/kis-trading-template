# 동적 손익비 멀티버스 — 8전략 전체 실행 결과 (측정 전용)

**핵심 질문**: 참조선(box/ATR/볼린저) 기반 **동적 sl/tp(손익비)** 가, 현재 라이브
전략들이 쓰는 **고정 sl/tp** 를 *강건하게* 이기는가?

**한 줄 결론**: **사실상 아니다.** 8전략 × 72개 동적 셀(총 576 동적 셀) 중 4관문
강건성 게이트를 통과한 셀은 **단 1개(book_pullback_ma5, 그것도 baseline Sharpe 0.07
의 사실상 깨진 전략을 이긴 것)** 뿐이다. 나머지 7전략은 **0개**. 고정 손익비는 이미
충분하며, 동적 손익비로 갈아탈 근거는 이번 측정에서 나오지 않았다.

### ★축소 사유 (정직 고지 — 이 결과의 최대 한계)
- 설계 의도는 **top_n=300 / boot_iters=500** 이었다. 그러나 셀당 풀기간 포트폴리오
  시뮬을 (풀기간 + train + test + 비용스트레스 base/cell + 부트스트랩) 다중 실행하므로
  **신호가 많을수록(elder/rs_leader 신호 1만~2만건) 전략당 4~6분**이 걸린다.
  top_n=8 예비런에서도 6전략에 ~8분이 걸려, top_n=300 풀런은 단일 세션에서
  완료 불가로 판단했다.
- 따라서 **top_n=40, boot_iters=150 으로 축소**하여 8전략을 모두 끝까지 완주시켰다.
  ("완벽보다 완주" 원칙 — 과제 지시.)
- **함의**: 유니버스가 대형주 40종목으로 좁고, 부트스트랩 분해능이 낮다(p05 추정
  분산↑). top_n=300 전체에서 결론이 바뀔 가능성은 배제할 수 없으나, **8전략 중
  7전략이 게이트의 *여러* 관문에서 큰 마진으로 탈락**(아래)했으므로 방향성 결론
  ("동적이 고정을 강건히 이기지 못한다")이 뒤집힐 가능성은 낮다고 본다. 단정은 보류.

### 운영 사고(인시던트) 고지
- 1차 런에서 `nohup ... &` 백그라운드 실행이 런처 셸 종료 후에도 살아남아
  (top_n=60 잔여 프로세스), 동시에 돌던 top_n=40 런과 **같은 출력 디렉토리에 교차
  기록**하여 `_summary.tsv` 와 grid 파일이 불일치하는 오염이 발생했다. 잔여
  프로세스를 종료하고 출력물을 **전량 삭제 후 단일 프로세스로 클린 재실행**하여
  현재 결과는 grid ↔ summary 가 전부 정합한다(8/8 baseline 일치 검증 완료).

## 4관문 강건성 게이트 (`evaluate_dynamic_gates`)

동적 셀이 "승리"로 인정되려면 **모두** 통과해야 한다:
1. **거래수** n_trades ≥ 30 **그리고** clamp_frac ≤ 0.20 (TP캡에 너무 자주 물리면 탈락)
2. **train 윈도우**: ΔSharpe > 0 **그리고** Sharpe > 0
3. **test 윈도우(OOS)**: ΔSharpe > 0 **그리고** Sharpe > 0
4. **부트스트랩**: ΔSharpe p05 > 0 (5분위가 양수 = 우연 아님)
5. **비용스트레스(+30bp)**: ΔSharpe > 0

`delta_sharpe` 등은 모두 **고정 베이스라인 대비** (cell_sharpe − base_sharpe).

## 유일한 통과 셀: book_pullback_ma5 (★ 강하게 의심할 것)

| 항목 | 값 |
|------|-----|
| 셀 | ref_type=**atr**, n=10, sl_mult=2.0, **RR=2.0** |
| baseline(고정) Sharpe | **0.074** (n_trades 762) |
| 셀 Sharpe (풀기간) | 0.635 |
| ΔSharpe (풀기간) | +0.561 |
| ΔSharpe (train) | +0.580 (Sharpe_train 0.579) |
| ΔSharpe (test/OOS) | +0.599 (Sharpe_test 1.194) |
| 부트스트랩 ΔSharpe **p05** | **+0.0026** (겨우 양수 — 0에 거의 붙음) |
| 비용스트레스(+30bp) ΔSharpe | +0.637 |
| n_trades / clamp_frac | 380 / 0.074 |

**정직한 해석**:
- 이 셀은 4관문을 형식적으로 모두 통과했고 train·test·비용에서 일관되게 양수다.
  그 점은 진짜 신호일 *가능성*을 시사한다.
- 그러나 **베이스라인이 Sharpe 0.074 = 사실상 깨진 고정 설정**(book_pullback_ma5는
  메모리상 OOS −21~97%로 부진하던 전략)이다. "동적이 고정을 이겼다"기보다 **"거의
  무엇으로든 0.07짜리 고정을 이긴다"**에 가깝다. 즉 *동적 손익비의 우월성*이 아니라
  *기존 고정 sl 3%/tp 15% 설정이 ma5에 부적합*함을 드러낸 것이다.
- 결정타: **부트스트랩 p05 = +0.0026** 으로 0에 거의 붙어 있다. boot_iters=150 으로
  **축소**된 점을 감안하면 이 p05의 부호는 매우 불안정하며, 더 많은 반복/넓은
  유니버스에서 음수로 뒤집힐 개연성이 충분하다. **단독 양성 1건을 "동적 손익비 채택
  근거"로 쓰면 안 된다** — 체리피킹 위험.

## 제외/스킵 고지

- **SKIP(<30 신호) 전략**: **없음.** 8전략 전부 top_n=40 에서 신호 ≥30 (최소
  deep_mr_dev20 149건)을 확보해 측정에 포함되었다.
- **clamp_frac>0.20 으로 제외된 셀**: 전략당 36~54개(위 표). 동적 그리드의 RR≥2
  고-TP 영역이 TP캡에 물려 대거 탈락. 이는 그리드 설계상 예상된 거름이며 은폐 없음.
- **ERROR**: 0건. 전 전략 정상완료.

## 종합 결론

> **동적 손익비(참조선 기반 sl/tp)가 고정 손익비를 *강건하게* 이긴 전략은
> 사실상 없다.** 8전략 576 동적 셀 중 4관문 통과 = 1건, 그마저 baseline 0.07짜리
> 깨진 고정설정을 이긴 것이고 부트스트랩 p05가 0에 붙어 있다.

- 7/8 전략에서 동적 셀은 OOS(test) 또는 부트스트랩 또는 TP캡 관문에서 광범위하게
  탈락했다. **고정 sl/tp 가 이미 적절하다**는 것이 이번 측정의 정직한 메시지다(이는
  완전히 유효한 결과 — 새 레버를 강제로 만들 필요 없음).
- 부수 발견(채택 권고 아님, **추가 측정 필요**): **book_pullback_ma5 의 *고정* sl/tp
  설정(3%/15%)이 ma5에 맞지 않을 수 있다.** 동적 손익비를 도입하라는 뜻이 아니라,
  이 전략의 *고정* 손익비를 재검토할 가치가 있다는 약한 신호다. 단, 그 결론조차
  top_n=300/boot=500 풀런으로 재확인하기 전에는 채택하지 말 것.
- **권고**: 동적 손익비 멀티버스는 **현 시점 채택 레버 없음**으로 종결.
  book_pullback_ma5 단서만 별도 후속(고정 손익비 재튜닝, 풀 유니버스)으로 이관.
