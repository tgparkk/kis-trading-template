## 1. 프로그램 시작 흐름

```
python main.py 실행
    ↓
┌─────────────────────────────────────┐
│ 1. 초기화 (initialize)              │
│  - API 연결                         │
│  - DB 연결                          │
│  - 텔레그램 봇 연결                  │
│  - 자금 관리자 초기화                │
│  - 전략 동적 로드                    │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 2. 상태 복원                        │
│  - DB에서 오늘 후보 종목 복원        │
│  - 보유 포지션 복원                  │
│    (수량, 매수가, 익절/손절률)       │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│ 3. 메인 루프 시작 (3개 태스크 병렬) │
│  - 메인 트레이딩 루프               │
│  - 시스템 모니터링 태스크            │
│  - 텔레그램 알림 태스크              │
└─────────────────────────────────────┘
```

## 2. 메인 루프 구조

프로그램은 **3개의 태스크**를 동시에 실행합니다:

### 🔄 동시 실행 태스크

| 태스크 | 역할 | critical | 비고 |
|--------|------|----------|------|
| **메인 트레이딩 루프** | 데이터수집→주문확인→포지션체크→매수판단→EOD | ✅ | 3초 간격 |
| **시스템 모니터링** | 스케줄 작업, 메모리/CPU 감시 | ❌ | 5초마다 체크 |
| **텔레그램** | 매매 알림 전송, 명령 수신 | ❌ | 이벤트 발생 시 |

### 메인 트레이딩 루프 (5단계 순차 실행)

```
while is_running and 장중:
  [1/5] 데이터 수집 (collect_once)
  [2/5] 미체결 주문 확인 (check_pending_orders_once)
  [3/5] 보유종목 체크 (현재가 업데이트 + 매도 판단)
  [4/5] 매수 판단 (매 3회 중 1회, ≈9초 간격)
  [5/5] EOD 일괄청산 체크
  → 3초 sleep
```

## 4. 매수 후보 종목 선별

### 🎯 전략 기반 종목 선별

매수 후보는 **전략의 `generate_signal()` 메서드**가 결정합니다. 프레임워크는 전략이 반환한 Signal에 따라 주문을 실행할 뿐, 종목 선별 로직 자체는 전략에 위임합니다.

```
┌──────────────────────────────────┐
│ 전략의 generate_signal() 호출    │
│  - 전략별 고유 로직 실행         │
│  - BUY / SELL / HOLD 판단       │
└──────────────────────────────────┘
        ↓
┌──────────────────────────────────┐
│ Signal 객체 반환                 │
│  - signal_type: BUY/SELL/HOLD   │
│  - confidence: 신뢰도 (0-100)   │
│  - target_price: 목표가          │
│  - stop_loss: 손절가             │
│  - reasons: 매수/매도 이유       │
└──────────────────────────────────┘
        ↓
    [TradingDecisionEngine이 해석]
        ↓
    [OrderManager가 주문 실행]
```

> 💡 **퀀트 전략 예시**: 퀀트 스크리닝 전략에서는 08:55에 8단계 팩터 점수(Value, Quality, Momentum, Size 등)를 계산하여 상위 30개 종목을 선정하고, 09:05 리밸런싱으로 포트폴리오를 재구성합니다.

## 5. 매수 판단

### 🛒 기본 매수 흐름

```
메인 트레이딩 루프 [4/5] 매수 판단
    ↓
전략의 generate_signal() 호출
    ↓
Signal.signal_type == BUY 또는 STRONG_BUY?
    ├─ 아니오 → 다음 루프로
    └─ 예 ↓
┌──────────────────────────────────┐
│ 매수 전 검증                      │
│  - 이미 보유 중인 종목인가?       │
│  - 충분한 자금이 있는가?          │
│  - 매수 쿨다운 중인가?           │
└──────────────────────────────────┘
    ↓ (검증 통과)
┌──────────────────────────────────┐
│ 주문 실행                         │
│  - Signal의 target_price, stop_loss 사용 │
│  - 시장가 또는 지정가 주문        │
│  - DB에 매매 기록 저장            │
└──────────────────────────────────┘
```

**핵심**: 전략이 **BUY Signal을 반환**하면, 프레임워크가 자금 확인 후 주문을 실행합니다. 매수 시점, 종목 선정 기준, 주문 방식 등은 모두 전략에 따라 다릅니다.

> 💡 **퀀트 전략 예시**: 09:05 리밸런싱 시 포트폴리오에 신규 편입된 종목을 시장가 매수하며, 종목 점수에 따라 차등 익절/손절률(S등급 20%/8%, A등급 17%/9% 등)을 설정합니다.

## 6. 매도 판단

### 💰 기본 매도 로직

매도는 두 가지 경로로 발생합니다:

#### 1. 전략 Signal 기반 매도

```
메인 트레이딩 루프 [3/5] 보유종목 체크
    ↓
각 보유 종목에 대해:
  ├─ 현재가 업데이트
  ├─ 전략의 generate_signal() 호출
  └─ Signal.signal_type == SELL?
      └─ 예 → 매도 주문 실행
```

#### 2. 기본 손절/익절 자동 매도

```
현재가 조회
    ↓
익절 조건 체크
    수익률 >= 목표 익절률?
    예) 매수가 10,000원, 익절률 20%
        → 현재가 12,000원 이상이면 매도
    ↓
손절 조건 체크
    수익률 <= -손절률?
    예) 매수가 10,000원, 손절률 10%
        → 현재가 9,000원 이하면 매도
    ↓
조건 만족 시 → 시장가 매도 주문
```

### 🕒 장마감 일괄청산 (선택사항)

15:00에 모든 보유 종목을 시장가로 일괄 매도 (데이 트레이딩 전략에서 사용)

## 📌 핵심 개념 요약

### 1. **전략 = 매매 로직의 핵심**
- `BaseStrategy`를 상속하여 `generate_signal()` 구현
- 프레임워크는 Signal에 따라 주문을 실행
- 전략만 교체하면 다양한 매매 방식 적용 가능

### 2. **장중 = 5단계 순차 실행**
- 3초마다 데이터수집 → 주문확인 → 포지션체크 → 매수판단 → EOD 체크
- 매도는 익절/손절 조건 또는 전략 Signal로 자동 실행

### 3. **데이터 수집 = 장 전 준비**
- 전략에 따라 장 시작 전 데이터 수집 (일봉, 재무 등)
- 과거 데이터 누적으로 분석 정확도 향상

### 4. **상태 복원 = 안전한 재시작**
- 프로그램 재시작 시 DB에서 보유 종목, 익절/손절률 자동 복원
- 중단 없이 모니터링 재개

## 🔧 설정 변경

### 전략 선택

`config.yaml` 또는 설정에서 전략명을 지정하면 `StrategyLoader`가 자동 로드:

```yaml
strategy:
  name: "sample"  # strategies/sample/ 폴더의 전략 사용
```

사용 가능한 전략: `sample`, `momentum`, `mean_reversion`, `volume_breakout`, `bb_reversion`, `lynch`, `sawkami` 등

### 익절/손절률

전략별로 Signal에 `target_price`, `stop_loss`를 포함하여 반환하거나, 기본값을 `config/constants.py`에서 설정합니다.

## ❓ 자주 묻는 질문

### Q1. 언제 매수하나요?
**A**: 전략의 `generate_signal()`이 **BUY Signal을 반환**할 때 매수합니다. 매수 시점은 전략에 따라 다릅니다.
> 예: 퀀트 전략에서는 09:05 리밸런싱 시 신규 편입 종목을 매수합니다.

### Q2. 언제 매도하나요?
**A**: 세 가지 경우:
- 전략이 SELL Signal을 반환할 때
- 익절가 도달 시 (3초마다 체크)
- 손절가 도달 시 (3초마다 체크)

### Q3. 종목은 어떻게 선정하나요?
**A**: 전략이 결정합니다. 각 전략은 고유한 종목 선정 로직을 가지고 있습니다.
> 예: 퀀트 전략은 8가지 팩터 점수로 상위 30개 선정, 모멘텀 전략은 연속 상승 종목 선정.

### Q4. 프로그램을 재시작하면?
**A**: DB에서 자동으로 보유 종목과 익절/손절률을 복원하여 모니터링 재개.

### Q5. 수동으로 개입할 수 있나요?
**A**: 네, 텔레그램 봇 명령으로 종목 추가/제거, 수동 매수/매도 가능.

### Q6. 새 전략을 만들려면?
**A**: `strategies/` 아래에 폴더를 만들고, `BaseStrategy`를 상속한 클래스에서 `generate_signal()`을 구현하면 됩니다. 상세는 [CLAUDE.md](CLAUDE.md)의 "전략 개발 방법" 섹션 참조.
