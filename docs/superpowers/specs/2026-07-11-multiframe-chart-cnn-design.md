# 멀티프레임 차트 CNN 선별기 — 설계

- **작성일**: 2026-07-11
- **브랜치(예정)**: `feat/multiframe-chart-cnn` (연구 전용, 라이브 무접촉)
- **전신**: `2026-07-09-intraday-rebound-discovery-design.md` — 워크포워드 사전등록 미달로 종료(gross +0.168% < 비용 0.21%). 본 설계는 그 종료 지점에서 **보유기간을 60분→3거래일로, 특징을 수치→차트 이미지로** 바꿔 경제성을 재구성한다.

---

## 1. 무엇을 만드는가

3·5·15분봉을 하나로 겹쳐 그린 **정규화된 차트 이미지**를 작은 CNN에 먹여, "지금 진입하면 **3거래일 안에** 목표가(TP)를 먼저 칠까 손절가(SL)를 먼저 칠까"를 판별하는 **종목 선별기**.

목적은 새 신호의 발명이 아니라 **선별로 거래당 gross 기대값을 끌어올리는 것**이다. 사장님 발제: "3·5·15분봉으로 패턴을 찾고, 진입 후 3거래일 이내 청산."

### 성패 기준선 (사후 조정 금지)

| 항목 | 값 | 근거 |
|---|---|---|
| 왕복비용 | 0.21% | `config/constants.py:118-119` 수수료 0.015%×2 + 거래세 0.18% |
| 통과 하한 (net) | > +0.05% | 직전 세션 사전등록과 동일 |
| 필요 gross | **≥ 0.26%** | 비용 0.21% + 통과선 0.05% |

직전 60분 창의 gross는 +0.168%였다. 3거래일로 늘려 TP를 5~10%대로 잡으면 **비용 0.21%가 상대적으로 작아진다** — 이것이 본 설계가 죽은 주제가 아닌 이유. 대신 오버나잇 갭·종가청산 리스크가 새로 생긴다.

---

## 2. 이미지 사양 (가설의 실체)

한 장의 이미지 = **"지금 이 종목을 살까?"를 판단하는 순간의 화면.**

### 시점 (표본 모집단)
- 유니버스: 상시수집 **199종목 고정**(`scripts/discovery/intraday_rebound/universe_snapshot.json`, 커버리지 ≥0.9). 데이터 편향(봇 스크리너 통과 종목 기록)은 제거 불가 → 적용범위 명시 한정.
- 후보 시점: **3분봉 매 마감 직후.** 09:00~10:00(룩백 부족) 제외, 15:30 마감 전 구간(전방창 확보 불가) 제외.
- 표본 규모: 199종목 × 356일 × 하루 ~80시점 ≈ **560만 장 규모.** (실측: kis_template.minute_candles 199종목 = 1분봉 23,269,360개 / 356일.) CNN 학습에 충분 → 급락 이벤트만 쓰면 수천 건이라 과적합 확실했던 문제를 무조건 표본이 해소.

### 그림 내용 — 6채널
3·5·15분봉 **각각 최근 60봉**을 **64×64**에 렌더. 프레임당 (가격봉, 거래량봉) 2채널 × 3프레임 = **6채널 텐서.**
- 3분봉 60봉 ≈ 3시간, 5분봉 ≈ 5시간, 15분봉 ≈ 2거래일. 같은 순간을 세 배율로 본다.
- **채널(가격)**: 캔들봉. 고가~저가 세로선 + 시가~종가 몸통. 양봉/음봉은 밝기값 구분(양봉 1.0, 음봉 0.5).
- **채널(거래량)**: 같은 가로 위치에 막대.

### 정규화 (사활)
- 각 채널 세로축은 **그 60봉 구간 자체의 고저/최대거래량으로 0~1 정규화.**
- 절대 가격·절대 거래량·**종목 정체성이 이미지에서 완전히 소거된다.** 삼성전자 7만원 차트와 잡주 3천원 차트가 같은 모양이면 CNN에게 같은 그림.
- 이걸 안 하면 CNN은 모양이 아니라 "이 종목은 X다 + X가 그 기간에 올랐다"를 외운다 → 백테스트 화려, 실전 사망. **본 프로젝트의 진짜 실패 지점.**

### 변동성 보조입력
- 정규화는 "구간이 얼마나 크게 움직였나"를 지운다. 60봉 안 3% 흔들린 차트와 15% 흔들린 차트가 같은 그림이 됨. 그런데 라벨(±TP/SL 선착)은 절대 변동성에 강하게 의존.
- 대응: **룩백 고저폭·ATR 두 스칼라를 이미지가 아닌 별도 입력으로 CNN 마지막 fully-connected 층에 concat.** 모양은 눈으로, 크기는 숫자로.

---

## 3. 라벨과 청산

- 진입가 = **다음 3분봉 시가.**
- 청산 구조 = **TP/SL 배리어 + 3거래일 강제청산:**
  - 3거래일 안에 TP 먼저 도달 → `up`(승)
  - SL 먼저 도달 → `down`(패)
  - 둘 다 미접촉 → 3일째 종가 청산 → `none`(실현손익 그대로 반영)
- **오버나잇 갭**: 다음날 시가로 실현(갭쓰루 반영, 낙관 금지).
- **TP/SL 값**: 학습구간에서만 3거래일 도달률을 실측해 결정. **평가구간은 절대 안 봄.** (직전 60분 연구엔 3일 도달률 데이터가 없음 → 감으로 정하지 않는다.)
- 구현: 직전 세션 `scripts/discovery/intraday_rebound/first_touch.py`를 **창만 60분→3거래일로 확장** 재사용. `labeler.py`도 동일.

## 4. 검증 (진짜 방어선)

직전 세션의 교훈: **아웃샘플을 하나만 쓰거나 격자를 흔들면 옳은 주제도 죽고 틀린 주제도 산다.** 방어 장치를 설계에 못 박는다.

### 워크포워드 사전등록
- TP/SL 규칙, CNN 구조, 통과조건(net>+0.05%, gross≥0.26%, 최소 거래빈도 ~1.47건/일)을 **결과 보기 전에 커밋.** 사전등록 커밋 해시가 증거.
- 확장창 다중 폴드, 각 폴드는 **학습구간에서만** TP/SL·정규화통계·CNN 가중치를 산출하고 **바로 다음 구간에서 한 번만** 평가. 평가구간은 선택에 미사용.
- 틀: 직전 `walkforward.py` 재사용.

### A→B 규칙 (격자탐색 방지)
- **A**: numpy 직접 래스터화 + 밑바닥 작은 CNN. PNG 안 만듦, 결정적, 테스트로 고정, 수백만 장 수분 생성.
- **B**: matplotlib PNG + ImageNet 사전학습 ResNet18 파인튜닝. (A 실패 시에만.)
- **핵심**: A의 성패와 B로 넘어갈지 여부는 **학습구간 내부 검증 성능으로만** 판정. **최종 워크포워드는 승자 한 모델에 대해 딱 한 번.** 두 번 시도해도 최종 검증은 한 번 → "여러 번 재면 우연히 통과" 메커니즘 차단.

### 누출 차단
- 정규화 통계·TP/SL·CNN 가중치·(있다면)클러스터 전부 **학습구간에서만** 산출.
- 이미지 렌더링은 **결정적 함수** → 입력 봉 → 픽셀이 테스트로 고정(직전 세션 재현 게이트가 우리를 구한 그 장치).
- 유효 표본은 봉이 아니라 **날짜.** 부트스트랩/폴드 경계 = 거래일.

### 운영 격리
- 연구 전용, **라이브 코드 무접촉.** 브랜치 격리. 학습은 GPU 사용, **장 마감 후** 실행(07:40 라이브 봇과 CPU/GPU 경쟁 회피).
- 환경: RTX 4080 SUPER 16GB 존재하나 **현재 torch 2.8.0+cpu(CUDA 미인식)** → CUDA 빌드 재설치 필요(블로커 아님, 학습시간 문제).

## 6. 알려진 리스크 / 미결

1. **디스크·시간 예산 미확정** — 560만 장 6채널 64×64 캐시 용량, 라벨러 실행시간(3거래일 창 × 199종목 × 356일)을 1차 스파이크로 실측 후 격자 확정.
2. **정규화가 변동성을 지운다** → 보조 스칼라로 보완하나, 그 보완이 충분한지는 실측 대상.
3. **오버나잇 갭 리스크** — 3일 보유는 종가·시가 갭에 노출. 라벨러가 갭을 다음날 시가로 정직하게 실현하는지 테스트 고정.
4. **CUDA 재설치** 후 결정성 확인(cudnn 비결정 연산 → seed 고정/deterministic 모드).
5. **A→B 유혹** — B까지 실패하면 종료. 세 번째 시도(다른 채널구성·다른 창) 착수 전 반드시 사전등록 재작성.
