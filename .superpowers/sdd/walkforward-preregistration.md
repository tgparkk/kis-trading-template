# 워크포워드 사전등록 (2026-07-10, 결과 보기 전에 커밋)

## 왜
15개월 전부를 이미 셀·특징·클러스터 선택에 소모했다. 깨끗한 OOS가 없다.
"형태를 더 만들어 규칙을 찾는" 일은 폴드 안에 가두어야만 정직해진다.

또한 블라인드 테스트는 검출력이 없다: n=120, 균형 60:60, 양측 alpha=0.05 에서
실제 적중률 55%(=우리가 아는 최대 신호를 사람이 완벽히 읽었을 때)의 검출력은 **15.7%**.
55% 를 검출하려면 약 790문항 필요. 그 테스트는 "사람이 알고리즘보다 압도적으로
잘 본다(>=65%)"만 잡을 수 있다. 의사결정 근거로 쓰지 않는다.

## 폴드
2025-04-01~2026-06-30 을 거래일 기준 8등분. 확장창(expanding):
  fold k: 학습 = fold 1..k, 평가 = fold k+1   (k=1..7 → 평가 7회)
평가 구간은 그 시점까지 어떤 선택에도 사용되지 않는다.

## 폴드 내 탐색 공간 (학습 구간에서만)
  TF in {3,5}
  D  in {0.04, 0.06, 0.08}
  theta_up in {0.02, 0.03, 0.04}, theta_dn in {0.02, 0.03}
  forward in {30, 60} 분
  필터 (택1):
    none
    close_pos_in_day <= q,  q in {0.2, 0.4}
    lower_wick_ratio <= q,  q in {0.2, 0.4}
    둘 다 (교집합), q=0.2
    cluster: KMeans(k in {6,8,10}) on z-normed pre20, 학습 edge_pp > 0 인 클러스터만 채택
             (평가 시 학습 중심점에 최근접 배정)
선택 기준: 학습 구간 net expectancy 최대. 제약: 학습 구간 평균 거래 >= 0.5건/일.
동점 시 거래수 많은 쪽.

## 비용 / 기대값
왕복 0.21% (수수료 0.015%x2 + 거래세 0.18%, config/constants.py:118-119)
gross = p_up*theta_up - (p_down + p_ambiguous)*theta_dn + p_none*mean_terminal_none
   (ambiguous = 같은 봉이 양 배리어 접촉 → 보수적으로 손절 처리)
net = gross - 0.0021
진입가 = 신호 봉 종가(낙관). 슬리피지 0(낙관). 이 두 가정은 결과에 유리하다.

## 중단 조건 (사후 조정 금지)
계속: 아래 셋을 모두 만족
  (1) 평가 7폴드 중 net > 0 인 폴드 >= 5
  (2) 전 폴드 합산(거래 가중) net >= +0.05% / 거래
  (3) 평가 구간 평균 거래 >= 1건 / 일
하나라도 미달 → 주제 종료. 손실회피 필터만 다른 전략에 이월.
"다른 각도"는 없다. 폴드 수·기간·격자·비용을 결과를 보고 바꾸지 않는다.
